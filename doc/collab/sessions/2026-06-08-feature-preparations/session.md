## 📑 Feature Analysis Summary

### 1. Mood Filtering (Fuzzy Logic & Radar UI)
* **Status**: Ready for implementation.
* **Document**: [mood_filtering_ideas.md](../../../proposals/mood_filtering_ideas.md)
* **Plan Path**: [prepare_mood_filtering.md](prepare_mood_filtering.md)
* **Main Findings**: 
  - Entirely frontend-driven. Since Essentia mood features are already stored in the DB and loaded client-side, similarity ranking can be executed at 60fps in the Svelte 5 store.
  - Svelte 5 store (`filters.svelte.ts`) will calculate membership using a trapezoidal function and compute weighted-OR/strict-AND combinations.
  - D3-based `MoodRadar.svelte` needs mouse handlers to support vertex dragging.

### 2. User Manual Song Editing & Overrides
* **Status**: Ready for implementation.
* **Document**: [user_edit_song.md](../../../proposals/user_edit_song.md)
* **Plan Path**: [prepare_user_edit_song.md](prepare_user_edit_song.md)
* **Main Findings**:
  - Requires a new migration (`33_user_track_overrides.sql`) adding the overrides table.
  - Rust query mappings in `database.rs` will be refactored to perform a `LEFT JOIN user_track_overrides` and map columns via `COALESCE(override.field, track.field)`.
  - Frontend edits will be managed via a dedicated `EditTrackModal.svelte`.

### 3. Persistent Map Layouts & Realignment
* **Status**: Ready for implementation.
* **Document**: [map_layouts.md](../../../proposals/map_layouts.md)
* **Plan Path**: [prepare_map_layouts.md](prepare_map_layouts.md)
* **Main Findings**:
  - Replaces PCA/UMAP on-demand calculations with cached layouts (`map_layouts` and `map_coordinates` tables).
  - Implements out-of-sample KNN projections in Rust to place new tracks without recalculating the entire layout.
  - Integrates 2D Orthogonal Procrustes alignment using SVD (`faer` crate) to normalize new projections against existing anchors, avoiding layout rotation/reflection drift.

### 4. Map Outliers & Topological Labeling
* **Status**: Ready for implementation.
* **Document**: [music_map_improvements.md](../../../proposals/music_map_improvements.md)
* **Plan Path**: [prepare_music_map_improvements.md](prepare_music_map_improvements.md)
* **Main Findings**:
  - Outliers identified via average $k$-NN distance are filtered and projected separately to prevent UMAP clustering compression.
  - Spatial regions are clustered using DBSCAN in Rust; text labels for regions are generated by computing TF-IDF scores on the tags of tracks in the cluster.
  - D3 force collision (`d3.forceCollide`) ensures labels do not overlap.

### 5. Playlist Drag & Reorder + TSP Optimizer
* **Status**: Ready for implementation.
* **Document**: [playlist_view_enhancements.md](../../../proposals/playlist_view_enhancements.md)
* **Plan Path**: [prepare_playlist_view_enhancements.md](prepare_playlist_view_enhancements.md)
* **Main Findings**:
  - Integrates HTML5 drag-and-drop rows in `TrackList.svelte` with optimistic UI updates.
  - Rust backend implements Traveling Salesperson Problem (TSP) solvers to minimize transition paths (Held-Karp exact solver for selections $\le 16$, 2-opt heuristic for larger lists).

### 6. SAX Structural Search
* **Status**: Ready for implementation.
* **Document**: [sax_structure.md](../../research/sax_structure.md)
* **Plan Path**: [prepare_sax_structural_search.md](prepare_sax_structural_search.md)
* **Main Findings**:
  - Performs string alignment searches over RMS envelope SAX codes (32 characters per track).
  - Replaces client-side JS edit-distance checks with a Rust-based Levenshtein Tauri IPC command (`search_by_structure`).
  - Outlines the layout for a frontend `BlockComposer.svelte` to filter tracks by structural patterns (e.g. Intro $\rightarrow$ Chorus).

### 7. Hum-to-Search (Query-by-Humming)
* **Status**: Ready for implementation.
* **Document**: [roadmap_ideas.md#8](../../../proposals/roadmap_ideas.md)
* **Plan Path**: [prepare_hum_to_search.md](prepare_hum_to_search.md)
* **Main Findings**:
  - Requires local CREPE pitch tracking ONNX inference during scan, storing time-series cents in a BLOB (`track_pitch_contours`) and 128-float summary vectors in a `vec0` table (`track_pitch_embeddings`).
  - Prunes database search via k-NN on summaries (Stage 1), then runs a subsequence DTW alignment in Rust to locate the hummed melody (Stage 2).
  - Frontend records audio using `getUserMedia` and sends raw buffers to Tauri.
